把个人网站做成「可对话的」并不是一个营销噱头。当你的花园里积累了几十万字的长文、上千条碎念，**检索本身就成了写作的延伸**：你在和过去的自己对话。这篇文章记录我把这座花园变成数字孪生的完整路径。

## 整体架构

先看数据流。内容的唯一真源（single source of truth）是 Git 仓库，其余一切都是派生物：

```
┌─────────────┐   push    ┌──────────────┐   embed   ┌──────────────┐
│  Git 仓库    │ ────────▶ │  CI 流水线    │ ────────▶ │   Postgres   │
│  (.md/.yml) │           │  diff→摘要    │           │  + pgvector  │
└─────────────┘           └──────────────┘           └──────┬───────┘
                                                            │ top-k
                          ┌──────────────┐   stream   ┌─────▼───────┐
                          │   访客浏览器  │ ◀───────── │  孪生 API    │
                          └──────────────┘    SSE     └─────────────┘
```

三条原则贯穿始终：

1. **内容不搬家**。Markdown 永远住在仓库里，Postgres 只存派生的摘要与向量。
2. **增量索引**。CI 只对 `git diff` 出的变更文件重新嵌入，全量重建是逃生舱而不是日常。
3. **降级可用**。向量库挂了，站点照常渲染；孪生 API 挂了，⌘K 退化为本地全文检索。

## 嵌入管道

嵌入这一步最容易被过度设计。我的版本只有 60 行，核心是按标题层级切块：

```ts
interface Chunk {
  path: string      // posts/digital-twin-rag.md
  heading: string   // "嵌入管道"
  content: string   // 该小节纯文本
  tokens: number
}

async function embedChanged(files: string[]) {
  const chunks = files.flatMap(splitByHeading)
  // 单条碎念天然就是一个 chunk，不再切分
  const batches = batch(chunks, 96)
  for (const b of batches) {
    const vectors = await embed(b.map(c => c.content))
    await upsert(b, vectors)
  }
}
```

几个实践中踩过的坑：

- 中文内容按 **字符数 × 0.6** 估算 token，比英文启发式准确得多；
- 标题要**拼接进 chunk 文本**（`# 嵌入管道\n\n...`），否则「管道怎么设计」这类问题召回不到；
- 碎念不要合并嵌入，它们的时间戳是重要的检索信号。

## 检索：混合比纯向量强

纯向量检索会漏掉命中稀有词的查询——比如用户问「Colemak 第几天了」，`colemak` 这个词在向量空间里毫无优势。解法是 BM25 与余弦距离的加权合成：

| 查询类型 | 纯向量 | 纯 BM25 | 混合 (0.6/0.4) |
| --- | --- | --- | --- |
| 语义改写（"怎么让博客会说话"） | ✅ 0.91 | ❌ 0.32 | ✅ 0.88 |
| 稀有词精确（"colemak 进度"） | ❌ 0.41 | ✅ 0.95 | ✅ 0.87 |
| 混合（"pgvector 调参心得"） | ⚠️ 0.68 | ⚠️ 0.71 | ✅ 0.90 |

> 数字是我自建的 40 条查询评测集上的 nDCG@5。样本不大，但趋势足够明确：**混合检索在两端都不吃亏**。

## 口吻对齐

孪生最难的不是检索，是「像我」。我的做法朴素但有效——人设 prompt 控制在 200 token 以内，只写风格约束不写事实（事实交给 RAG）：

```text
你是这座花园主人的数字孪生。规则：
- 用检索到的片段回答，没有依据时明说"花园里还没写过"
- 语气：直接、具体、偶尔自嘲，避免"作为一个AI"式套话
- 引用原文时给出 [标题](路径) 形式的链接
- 中文提问用中文答，英文提问用英文答
```

温度设 0.4。再高会开始「演」，再低就成了搜索结果朗读机。

## 成本

很多人以为这套东西很贵。实际账单（月均）：

- 嵌入：增量更新，**不到 $0.5**
- 对话：中等流量下 $3–8，缓存命中后更低
- Postgres：与站点共用一个实例，边际成本为零

结论：一台小 VPS + 一个推理 API key，就能养一个孪生。剩下的都是写作本身。
