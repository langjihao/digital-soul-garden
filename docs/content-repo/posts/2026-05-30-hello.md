欢迎来到我的数字花园。这里的每一篇文章都是一个 markdown 文件，住在 GitHub 私有仓库。

## 它怎么工作

- `git push` 触发 GitHub Actions
- Action 把事件签名后 POST 到站点 `/api/public/ingest`
- 站点解析、切片、向量化，写入 Postgres
- 你在 `⌘K` 里搜索的就是这些向量

## 为什么这么干

因为我不想被任何 SaaS 锁定，也不想再写"所见即所得"的编辑器。Git 是最好的版本管理，PR 是最好的审阅流程，Markdown 是最好的写作格式。
